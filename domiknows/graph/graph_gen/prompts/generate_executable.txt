## Question Type → Constraint Pattern

### Existence: "Is there a [property] [object]?"
→ `execute(existsL(andL(property('x'), object('x'))))`
Example: "Is there a red cube?"
→ `execute(existsL(andL(red('x'), cube('x'))))`

### Relation: "Is X [relation] Y?"
→ `execute(existsL(relation(iotaL(X_description), iotaL(Y_description))))`
Example: "Is the red cube left of the blue sphere?"
→ `execute(existsL(left_of(iotaL(andL(red('x'), cube('x'))), iotaL(andL(blue('y'), sphere('y'))))))`

### Counting: "How many [property] [objects]...?" (answer = N)
→ `execute(exactL(andL(property('x'), object('x')), N))`
Example: "How many red cubes are there?" (answer = N)
→ `execute(exactL(andL(red('x'), cube('x')), N))`

### "Are there at least/more than N...?"
→ `execute(atLeastL(property_filter('x'), N))`
Example: "Are there at least 3 metal objects?"
→ `execute(atLeastL(andL(metal('x'), object('x')), 3))`

### Comparative: "Are there more X than Y?"
→ `execute(greaterL(X_filter('x'), Y_filter('y')))`
Example: "Are there more cubes than spheres?"
→ `execute(greaterL(cube('x'), sphere('y')))`

### Attribute Query: "What [attribute] is the [description]?"
→ `execute(queryL(attribute_concept, iotaL(description_filter)))`
Example: "What color is the large cylinder?"
→ `execute(queryL(color, iotaL(andL(large('x'), cylinder('x')))))`

### Counting: "How many objects with the given property exist"
→ `execute(sumL(property_filter('x')))`
Example: "How many red cubes are there?" (answer = N)
→ `execute(sumL(andL(red('x'), cube('x'))))`

## Available Constraint Operators

### Boolean Logic
- `andL(a, b)` — a AND b
- `orL(a, b)` — a OR b
- `notL(a)` — NOT a
- `ifL(a, b)` — a → b (implication)
- `nandL(a, b)` — NOT(a AND b)

### Element-wise Counting (within a single sample)
- `existsL(a)` — at least 1 entity satisfies (∃x)
- `atLeastL(a, k)` — count ≥ k
- `atMostL(a, k)` — count ≤ k
- `exactL(a, k)` — count == k

### Comparative Counting (compare counts between two sets)
- `greaterL(a, b)` — count(a) > count(b)
- `lessL(a, b)` — count(a) < count(b)
- `equalCountsL(a, b)` — count(a) == count(b)
- 'sumL(a)' — count(a) == N

### Entity Selection — iotaL (Definite Description)
`iotaL` selects THE unique entity satisfying a condition.
- `iotaL(andL(red('x'), cube('x')))` — select THE red cube
- Can be nested: `left_of('x', iotaL(andL(blue('y'), cube('y'))))`

### Attribute Query — queryL
`queryL(attribute_concept, iotaL(entity_description))` — "What is the <attribute> of THE <entity>?"
- First argument: the PARENT concept (with `is_a()` children or EnumConcept).
- Second argument: typically an `iotaL(...)` selecting the entity.
For `queryL`, use the PARENT concept as the first argument:
```python
queryL(color, iotaL(big('x')))   # "What color is the big thing?"
```

## How Relations Work — has_a and Path Expressions

Binary relations are defined with `has_a()`:
```python
left_of = Concept('left_of')
(rel_left_of_obj1, rel_left_of_obj2) = left_of.has_a(object, object)
```

In constraints, relations take variable arguments directly (just like concepts):
```python
left_of('x', 'y')   # "x is left_of y"
```

How to build the `execute()` call for a given question:
1. Identify the question type (Existence, Relation, Counting, Comparative, Attribute Query).
2. Use the corresponding constraint pattern from above.
3. Identify how many objects are mentioned in the question and create variables for them (e.g., 'x', 'y', 'z').
4. For each object, identify its properties and relations to other objects.
5. If there is only single object realtion are not needed in this case, just use the properties of the object in the constraint.
6. If two objects are mentioned, identify the relation between them and use the relation in the constraint. Ususly  single relation is needed in this case.
7. If there are more than two objects, identify the relations between them and use them in the constraint. Ususly 2-3 relations are needed in this case.
8. Add comments after each element of the constraint to indicate which part of the question it corresponds to (e.g., `# big object`, `# right of brown cylinder`).
